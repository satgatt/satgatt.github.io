미션 목표

문제 목표 :  
홈 디렉토리의 readme 파일에서 다음 레벨 비밀번호를 찾기.

해결 방법 :

cat readme 명령으로 파일 내용을 출력해 비밀번호를 얻는다.  
exit 으로 bandit0 세션을 종료한 뒤, ssh bandit1@bandit.labs.overthewire.org -p 2220 으로 bandit1에 접속하고 얻은 비밀번호를 입력한다.  

사용 명령어 정리 :  
ls : 현재 디렉토리의 파일/폴더 목록 출력  
cat : 파일 내용을 터미널에 출력  
exit : 현재 SSH 세션 종료

사용 명령어 정리 :  
./파일명 : 현재 디렉토리의 파일임을 명시 (특수 문자로 시작하는 파일명 다룰 때 사용)

문제 목표 :  
홈 디렉토리의 --spaces in this filename-- 파일에서 다음 레벨 비밀번호를 찾기.  
주어진 내용 :  
파일명이 -- 로 시작하고 공백도 포함되어 있음. cat 명령어는 -- 로 시작하는 인자를 옵션으로 해석하기 때문에 그대로 입력하면 unrecognized option 에러가 발생함.  
해결 방법 :  
ssh bandit2@bandit.labs.overthewire.org -p 2220 으로 SSH 접속한다.  
공백 처리를 위해 파일명 전체를 따옴표로 감싼다.  
-- 로 시작하는 문제를 해결하기 위해 ./ 를 앞에 붙여 경로를 명시한다.  
최종적으로 cat "./--spaces in this filename--" 실행하면 비밀번호가 출력된다.  

사용 명령어 정리 :  
./파일명 : 현재 디렉토리의 파일임을 명시. - 나 -- 로 시작하는 파일명을 옵션으로 오해하지 않게 함  
-- (인자로서) : "이후로는 옵션이 아닌 파일명/인자이다" 를 알리는 구분자

<img width="1115" height="628" alt="image" src="https://github.com/user-attachments/assets/c9f055e7-0205-4879-9dd0-54ff7a9cdfee" />

문제 목표 :  
inhere 하위에서 텍스트 파일이면서, 크기가 1033 바이트이고, 실행 불가능한 파일을 찾아 비밀번호 확인하기.  
주어진 내용 :  
inhere 안에 maybehere00 ~ maybehere19 까지 20개 폴더가 있고 각 폴더에 여러 파일이 들어있음. 수동 탐색은 비효율적이라 조건 검색이 필요함.  
해결 방법 :  
ssh bandit5@bandit.labs.overthewire.org -p 2220 으로 SSH 접속한다.  
cd inhere 로 이동한다.  
find . -type f -size 1033c ! -executable 명령으로 조건을 모두 만족하는 파일을 한 번에 검색한다.  
출력된 경로를 cat 으로 열어 비밀번호를 얻는다.  

사용 명령어 정리 :  
find 경로 [조건] : 지정한 경로 아래에서 조건에 맞는 파일/폴더를 검색  
-type f : 일반 파일만 검색 (d 는 디렉토리)  
-size 1033c : 파일 크기 조건. c = byte, k = KB, M = MB  
! 조건 : 조건의 부정 (NOT 연산)  
-executable : 실행 가능한 파일만 검색하는 조건

서버 전체에서 소유자 bandit7, 그룹 bandit6, 크기 33 바이트인 파일을 찾아 비밀번호 확인하기.

2>/dev/null : 표준 에러 출력(stderr, 파일 디스크립터 2번) 을 /dev/null (빈 공간) 로 보내서 화면에 안 보이게 함

| (파이프) : 앞 명령의 출력을 뒷 명령의 입력으로 연결. 명령어 조합의 핵심

### level 11 > 12

<img width="1115" height="628" alt="image" src="https://github.com/user-attachments/assets/13b7b8aa-d781-4c25-b498-26b67f1f0d59" />

문제 목표 :  
ROT13 으로 인코딩된 data.txt 를 복호화해서 비밀번호 확인하기.  
주어진 내용 :  
data.txt 의 알파벳이 ROT13 방식으로 13글자씩 이동되어 있음. ROT13은 인코딩과 디코딩이 동일한 자기역(self-inverse) 변환이다.  
해결 방법 :  
ssh bandit11@bandit.labs.overthewire.org -p 2220 으로 SSH 접속한다.  
cat data.txt | tr 'A-Za-z' 'N-ZA-Mn-za-m' 명령으로 알파벳을 13글자씩 되돌린다.  
대문자 A-Z → N-ZA-M, 소문자 a-z → n-za-m 로 매핑해서 원본 텍스트를 복원하고 비밀번호를 얻는다.

문제 목표 :         
localhost:30002 데몬에 bandit24 비밀번호와 4자리 PIN(0000~9999)을 무차별 대입으로 시도해 bandit25 비밀번호 얻기.            
주어진 내용 :             
데몬은 bandit24 비밀번호와 4자리 PIN을 함께 받아야 응답한다. PIN을 알 방법이 없으므로 10000개 조합을 모두 시도하는 brute-force가 필요하다. 단일 연결에서 여러 시도가 가능하므로 모든 조합을 미리 만들어 한 번에 전송할 수 있다.  
해결 방법 :     
ssh bandit24@bandit.labs.overthewire.org -p 2220 으로 SSH 접속한다.                            
mkdir /tmp/bf24 && cd /tmp/bf24 로 작업 공간을 만든다.   
PASS=$(cat /etc/bandit_pass/bandit24) 로 비밀번호를 변수에 저장한다.         
for i in {0000..9999}; do echo "$PASS $i"; done > attempts.txt 명령으로 0000부터 9999까지 PIN과 비밀번호를 조합한 10000줄짜리 입력 파일을 만든다.               
cat attempts.txt | nc localhost 30002 > results.txt 명령으로 모든 조합을 데몬에 한 번에 전송하고 응답을 파일에 저장한다.            
grep -v Wrong results.txt 로 실패 메시지를 제외한 줄을 출력해 bandit25 비밀번호를 확인한다.             
사용 명령어 정리 :          
        
{시작..끝} : bash brace expansion. 연속된 숫자/문자 자동 생성. {0000..9999} 는 0 패딩 포함         
for 변수 in 목록; do 명령; done : 쉘 반복문         
명령 > 파일 : 표준 출력을 파일에 저장 (덮어쓰기)           
grep -v 패턴 : 패턴이 포함되지 않은 줄만 출력 (invert match)                   
무차별 대입 공격(brute force) : 가능한 모든 조합을 시도하는 단순 무식한 공격 방법
