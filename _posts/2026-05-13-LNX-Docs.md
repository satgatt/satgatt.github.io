# 리눅스 기반 시스템 탐색 및 데이터 분석

미션 목표  
WSL(Windows Subsystem for Linux) 환경에서 리눅스 기본 명령어를 활용하여 파일 시스템을 탐색하고,    
숨김 파일 및 특수 파일을 식별하며, 인코딩된 데이터를 해석하는 능력을 습득한다.    
또한CLI(Command Line Interface) 환경에 익숙해지고, 보안 및 시스템 분석의 기초 역량을 기른다.

-p 2220 : 포트 번호 지정  
비밀번호 bandit0 입력하면 로그인 완료.  

### level 0 > 1

<img width="1060" height="685" alt="image" src="https://github.com/user-attachments/assets/1bfac26c-a274-4b11-bc8a-4fe3e33d8eb5" />

문제 목표 :  
홈 디렉토리의 readme 파일에서 다음 레벨 비밀번호를 찾기.  
주어진 내용 :  
홈 디렉토리에 readme 라는 파일이 있고, 그 안에 bandit1의 비밀번호가 저장되어 있음.  
해결 방법 :  
ls 로 홈 디렉토리 파일 목록을 확인해서 readme 파일이 있는지 확인한다.  
cat readme 명령으로 파일 내용을 출력해 비밀번호를 얻는다.  
exit 으로 bandit0 세션을 종료한 뒤, ssh bandit1@bandit.labs.overthewire.org -p 2220 으로 bandit1에 접속하고 얻은 비밀번호를 입력한다.  

사용 명령어 정리 :  
ls : 현재 디렉토리의 파일/폴더 목록 출력  
cat : 파일 내용을 터미널에 출력  
exit : 현재 SSH 세션 종료  

### level 1 > 2

<img width="1060" height="685" alt="image" src="https://github.com/user-attachments/assets/127c94e7-52d9-45a4-88bd-f99907ca7661" />


문제 목표 :  
홈 디렉토리의 - 라는 이름의 파일에서 다음 레벨 비밀번호를 찾기.  
주어진 내용 :  
파일명이 - (하이픈) 이라서 일반적인 방법으로 열리지 않음. Linux에서 - 는 표준 입력을 가리키는 특수 문자로 쓰이기 때문.  
해결 방법 :  
ls 로 파일 존재를 확인한다. 
cat ./- 처럼 상대 경로 ./ 를 붙여서 명령어가 - 를 파일명으로 인식하게 한다.  

사용 명령어 정리 :  
./파일명 : 현재 디렉토리의 파일임을 명시 (특수 문자로 시작하는 파일명 다룰 때 사용)
