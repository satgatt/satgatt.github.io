# 리눅스 기반 시스템 탐색 및 데이터 분석

미션 목표  
WSL(Windows Subsystem for Linux) 환경에서 리눅스 기본 명령어를 활용하여 파일 시스템을 탐색하고,    
숨김 파일 및 특수 파일을 식별하며, 인코딩된 데이터를 해석하는 능력을 습득한다.    
또한CLI(Command Line Interface) 환경에 익숙해지고, 보안 및 시스템 분석의 기초 역량을 기른다.

### level 0 > 1

<img width="1060" height="685" alt="image" src="https://github.com/user-attachments/assets/1bfac26c-a274-4b11-bc8a-4fe3e33d8eb5" />

문제 목표 :  
홈 디렉토리의 readme 파일에서 다음 레벨 비밀번호를 찾기.  
주어진 내용 :  
홈 디렉토리에 readme 라는 파일이 있고, 그 안에 bandit1의 비밀번호가 저장되어 있음.  
해결 방법 :  
ls 로 홈 디렉토리 파일 목록을 확인해서 readme 파일이 있는지 확인한다.  
cat readme 명령으로 파일 내용을 출력해 비밀번호를 얻는다.  
exit 으로 bandit0 세션을 종료한 뒤, ssh bandit1@bandit.labs.overthewire.org -p 2220 으로 bandit1에 접속하고 얻은 비밀번호를 입력한다.  

사용 명령어 정리 :  
ls : 현재 디렉토리의 파일/폴더 목록 출력  
cat : 파일 내용을 터미널에 출력  
exit : 현재 SSH 세션 종료  

### level 1 > 2

<img width="1060" height="685" alt="image" src="https://github.com/user-attachments/assets/127c94e7-52d9-45a4-88bd-f99907ca7661" />


문제 목표 :  
홈 디렉토리의 - 라는 이름의 파일에서 다음 레벨 비밀번호를 찾기.  
주어진 내용 :  
파일명이 - (하이픈) 이라서 일반적인 방법으로 열리지 않음. Linux에서 - 는 표준 입력을 가리키는 특수 문자로 쓰이기 때문.  
해결 방법 :  
ls 로 파일 존재를 확인한다. 
cat ./- 처럼 상대 경로 ./ 를 붙여서 명령어가 - 를 파일명으로 인식하게 한다.  

사용 명령어 정리 :  
./파일명 : 현재 디렉토리의 파일임을 명시 (특수 문자로 시작하는 파일명 다룰 때 사용)  

### lever 2 > 3

<img width="1115" height="628" alt="image" src="https://github.com/user-attachments/assets/d745b03c-71f2-4d16-916c-0ce23d01e7c9" />

문제 목표 :  
홈 디렉토리의 --spaces in this filename-- 파일에서 다음 레벨 비밀번호를 찾기.  
주어진 내용 :  
파일명이 -- 로 시작하고 공백도 포함되어 있음. cat 명령어는 -- 로 시작하는 인자를 옵션으로 해석하기 때문에 그대로 입력하면 unrecognized option 에러가 발생함.  
해결 방법 :  
ssh bandit2@bandit.labs.overthewire.org -p 2220 으로 SSH 접속한다.  
공백 처리를 위해 파일명 전체를 따옴표로 감싼다.  
-- 로 시작하는 문제를 해결하기 위해 ./ 를 앞에 붙여 경로를 명시한다.  
최종적으로 cat "./--spaces in this filename--" 실행하면 비밀번호가 출력된다.  

사용 명령어 정리 :  
./파일명 : 현재 디렉토리의 파일임을 명시. - 나 -- 로 시작하는 파일명을 옵션으로 오해하지 않게 함  
-- (인자로서) : "이후로는 옵션이 아닌 파일명/인자이다" 를 알리는 구분자  


### level 3 > 4

<img width="1115" height="628" alt="image" src="https://github.com/user-attachments/assets/80e0441a-0e50-4016-82cd-dd9ade80d770" />

문제 목표 :  
inhere 디렉토리 안의 숨김 파일에서 다음 레벨 비밀번호를 찾기.  
주어진 내용 :  
비밀번호가 숨김 파일에 저장되어 있음. Linux에서 파일명이 . 으로 시작하면 숨김 파일로 분류되어 기본 ls 명령으로는 보이지 않음.  
해결 방법 :  
ssh bandit3@bandit.labs.overthewire.org -p 2220 으로 SSH 접속한다.  
cd inhere 로 폴더 이동한다.   
ls -a 명령으로 숨김 파일까지 모두 표시한다.  
보이는 숨김 파일을 cat 으로 출력해 비밀번호를 얻는다.  

사용 명령어 정리 :
cd 디렉토리명 : 해당 디렉토리로 이동  
ls -a : 숨김 파일(. 으로 시작하는 파일) 포함 모든 파일 표시. a 는 "all" 의 약자  

### level 4 > 5

<img width="1115" height="628" alt="image" src="https://github.com/user-attachments/assets/3ee044d6-592a-42e1-a7d2-19117698ee3e" />

문제 목표 :  
inhere 디렉토리 안의 여러 파일 중 사람이 읽을 수 있는 텍스트 파일을 찾아 비밀번호를 확인하기.  
주어진 내용 :  
디렉토리 안에 10개 파일이 있고 대부분 바이너리 데이터. 그중 하나만 텍스트 파일이며 비밀번호가 들어있음. 파일명이 - 로 시작해서 처리에 주의가 필요함.  
해결 방법 :  
ssh bandit4@bandit.labs.overthewire.org -p 2220 으로 SSH 접속한다.   
cd inhere 로 폴더 이동 후 ls 로 파일 목록을 확인한다.  
file ./* 명령으로 모든 파일의 타입을 한 번에 확인한다. * 는 모든 파일을 의미 ./ 를 붙여 - 로 시작하는 파일명 문제를 회피한다.  
결과에서 ASCII text 로 표시된 파일을 cat ./파일명 으로 출력해 비밀번호를 얻는다.  

사용 명령어 정리 :  
file 파일명 : 파일의 종류(텍스트/바이너리/이미지 등) 를 판별  
* : 모든 파일/문자를 의미하는 와일드카드. file ./* 는 현재 디렉토리의 모든 파일에 명령 적용  

### level 5 > 6

<img width="1115" height="628" alt="image" src="https://github.com/user-attachments/assets/c9f055e7-0205-4879-9dd0-54ff7a9cdfee" />

문제 목표 :  
inhere 하위에서 텍스트 파일이면서, 크기가 1033 바이트이고, 실행 불가능한 파일을 찾아 비밀번호 확인하기.  
주어진 내용 :  
inhere 안에 maybehere00 ~ maybehere19 까지 20개 폴더가 있고 각 폴더에 여러 파일이 들어있음. 수동 탐색은 비효율적이라 조건 검색이 필요함.  
해결 방법 :  
ssh bandit5@bandit.labs.overthewire.org -p 2220 으로 SSH 접속한다.  
cd inhere 로 이동한다.  
find . -type f -size 1033c ! -executable 명령으로 조건을 모두 만족하는 파일을 한 번에 검색한다.  
출력된 경로를 cat 으로 열어 비밀번호를 얻는다.  

사용 명령어 정리 :  
find 경로 [조건] : 지정한 경로 아래에서 조건에 맞는 파일/폴더를 검색  
-type f : 일반 파일만 검색 (d 는 디렉토리)  
-size 1033c : 파일 크기 조건. c = byte, k = KB, M = MB  
! 조건 : 조건의 부정 (NOT 연산)  
-executable : 실행 가능한 파일만 검색하는 조건
